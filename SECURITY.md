# Security Policy

## Reporting a Vulnerability

This repo is a markdown catalog with no executable code. Genuine "security" issues are rare here, but if you find one (e.g. a malicious link, a typosquatting concern, a doc that could induce harm), please report it privately rather than opening a public issue.

**How to report:**

- **Preferred**: GitHub's [private vulnerability reporting](https://github.com/christophecapel/claude-mechanisms/security/advisories/new)
- **Alternative**: DM [@christophecapel](https://github.com/christophecapel) on GitHub

For the paired toolkit's security (`hooks/`, `skills/`, `lib/`), see [claude-mechanisms-tools/SECURITY.md](https://github.com/christophecapel/claude-mechanisms-tools/blob/main/SECURITY.md).

## Response timeline

- **Acknowledgment**: within 7 days
- **Initial assessment**: within 14 days
- **Fix**: aimed for 90 days from acknowledgment

## Disclosure

After a fix lands, the vulnerability is disclosed via a GitHub Security Advisory and a `CHANGELOG.md` note. Reporters are credited (with permission).

No bug bounty program — this is a personal-scale catalog. Reports are appreciated regardless.
